Add Managed Connection

DataDock will authenticate as the OS account it's running under. This only works when the DataDock server process itself runs on Windows.
Accepts SQL Server URL or ADO (Server=...;User Id=...;Password=...;) connection strings, PostgreSQL/MySQL URLs, or a SQLite file path.
This adds the connection only for you — no one else on this server sees or can use it, nothing is written to disk, and it's gone on restart. Host/port/database/user are remembered in this browser for next time (never the password). To share a connection with everyone (and let an admin make it the default), an admin can do so from the list above after logging in.

Auto-Detect Sources

DataDock also checks DATABASE_URL, POSTGRES_URL, MYSQL_URL, MARIADB_URL, MSSQL_URL, SQLITE_PATH, and DATADOCK_CONNECTIONS at startup. DATADOCK_CONNECTIONS accepts a JSON array with id, name, kind, dsn, and default. When "Type" is left on "Auto detect", pasted SQL Server ADO connection strings (Server=...;User Id=...;Password=...;) are recognized automatically, alongside PostgreSQL/MySQL/SQL Server URLs and SQLite paths.
