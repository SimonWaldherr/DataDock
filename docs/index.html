Why DataDock

Not another
pretty shell.

Most SQL clients start once your data is clean, connected, and modeled. DataDock starts before that, when the files, formats, and relationships are still fighting you.

01 / IMPORT

Every file gets a seat at the table.

CSV, JSON, XLSX, SQLite, GeoJSON, KML, OSM, PMTiles and more land in one queryable local workspace.

02 / SEE

SQL results that do more than rows.

Move from result grids to charts, maps, routes, data sheets, quality reports, and export-ready data without changing tools.

03 / CONNECT

Local-first. Multi-database when needed.

Begin with embedded tinySQL, then manage SQLite, PostgreSQL, MySQL/MariaDB, and SQL Server connections in the same cockpit.

04 / MATCH

Find the same thing, spelled differently.

Turn inconsistent customer, address, article, and reference data into confirmed, queryable crosswalks.

05 / ASK

AI that knows the schema, not your secrets.

Optionally bring an OpenAI-compatible model for SQL drafting, explanations, and grounded schema context.

06 / OWN

Your workflow is not a SaaS dashboard.

One Go process, local files, portable snapshots, controlled credentials, and an open source codebase you can inspect.

07 / BROWSE

Inspect, sort, page, and change real records.

Use datasheets, visual table design, CRUD for id-backed tables, catalog refreshes, and simple table lifecycle controls.

08 / SHAPE

Turn query output into the right view.

Switch an editor result among table, cards, JSON/XML trees, pivots, column profiles, schema graph, notebook, live log, chart, or map.

09 / REMEMBER

Keep the useful questions close.

Run selected SQL with Ctrl+Enter or F5, retain browser-local query history, export it, and share a URL that restores the query and view.

10 / PROVE

Know where the data came from.

Every local/API import carries source hash, size, format, row count, geometry validity, type, and WGS84-bounds reporting.

11 / SCHEDULE

Make repeat work less repetitive.

Use the local scheduler for registered jobs, manual runs, and a bundled demo workflow without operating another service.

12 / PROTECT

Operate deliberately.

Roles, maintenance mode, audit logging, read-only operation, and encrypted disk-backed tinySQL modes keep control with the operator.

Capability map

The whole
data journey.

DataDock is not a query box with a connection list. It is a local workbench for turning unfamiliar inputs into inspected, spatially-aware, portable results.

Bring it in

Files without a detour.

Normalize common office, structured, binary, archive, and map formats into tables. Keep columnar files as deliberate manifests when a bundled row decoder is not available.

  • CSV and TSV with UTF-8/UTF-16 BOM handling
  • JSON, NDJSON, YAML, XML, HTML tables
  • Multi-sheet XLSX and SQLite tables/views
  • MessagePack, CBOR, BSON, iCalendar, vCard
  • Parquet, Arrow, Feather, DuckDB manifests
  • Watch a directory for supported file updates
Work the map

Spatial data is first-class data.

Bring geometry into SQL, serve local raster or vector tiles, and take routing questions directly to GeoJSON output instead of exporting to another GIS first.

  • GeoJSON, GeoPackage, GPX, KML, OSM XML/PBF
  • Shapefile ZIP, MBTiles, PMTiles v3
  • TileJSON and XYZ tile endpoints
  • Shortest path and reachable-area analysis
  • GEO_DISTANCE and GEO_WITHIN_BBOX SQL
  • Spatial quality, geometry, and bounds reports
Query and reconcile

One SQL desk, many sources.

Start in embedded tinySQL, switch the active connection per browser session, and carry useful work across engines without losing the local data workbench.

  • tinySQL, SQLite, PostgreSQL, MySQL/MariaDB, SQL Server
  • Monaco SQL editor with fallback editor
  • Examples, prompt starters, shareable query URLs
  • Cross-connection table migration
  • Domain-agnostic entity matching and review
  • Save matches as queryable crosswalks
Export and operate

Answers that travel well.

Turn a full table or a result set into the next system’s preferred format, while keeping the server’s security controls and operational state visible to its owners.

  • CSV, Excel-safe CSV, TSV, XLSX, JSON, NDJSON
  • XML, HTML, SQLite, GeoJSON, KML, GPX, Shapefile ZIP
  • GeoJSON simplify, bbox, fields, and explode options
  • Admin roles, maintenance mode, and audit log
  • Portable snapshots and read-only serving
  • AES-256 encrypted disk, JSON, hybrid, and index modes

Table craft

Browse, paginate, sort, create, edit, delete. Datasheets and a visual table designer make everyday database work available without leaving the browser.

Result craft

Table, cards, chart, map, pivot, profile, graph, notebook. Select the representation that helps answer the actual question.

Agent craft

Optional OpenAI-compatible assistance. Draft SQL, explain results or errors, and inspect the same dialect-aware schema context it receives.

tinySQL craft

Vector search, native geo functions, index catalog, and agent context. Modern local capabilities without a separate database service.

No black box

Power, with
operator control.

DataDock keeps sophisticated features optional and visible: local data can stay local, administrators can control changes, and automation has stable HTTP and data standards to work with.

Use VEC_SEARCH for repeated local retrieval with a bounded, vector-free cache. Keep ANN HNSW opt-in and warm it after a bulk reindex, never on routine ingestion. Snapshot export is portable; restore remains an explicit operator responsibility.
